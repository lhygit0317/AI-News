# 超聚变(xFusion)多维度情报系统 — 架构设计文档

> **版本**: 1.0  
> **创建日期**: 2026-06-27  
> **基于**: `nomagicln/hermes-assets` 的 `server-industry-weekly-intelligence` 技能

---

## 1. 设计目标

为超聚变(xFusion)构建一套**自动化行业情报收集系统**，覆盖五大业务维度：

| 维度 | 目标 | 数据源 |
|------|------|--------|
| 电源+液冷 | 监控服务器电源生态和液冷散热技术发展 | Google/Bing News, ServeTheHome, DataCenterDynamics |
| AI大模型 | 跟踪主流大模型厂商动态，判断算力需求走势 | Google News, 机器之心, TechCrunch |
| AI基础设施方案 | 监控竞品和合作伙伴的方案策略 | Google News, SCMP, Nikkei Asia |
| 太空算力 | 前瞻性监控太空计算方向 | Google News, NASA, AWS Space |
| 人力资源 | 竞品招聘和人才流动情报 | 脉脉, 猎聘, LinkedIn, Google News |

### 分层设计

| 层级 | 组件 | 职责 |
|------|------|------|
| **Skill 层** | SKILL.md + sub-skills/ | 定义执行流程、搜索策略、输出格式 |
| **Agent 层** | delegate_task() | 并行执行情报采集子代理 |
| **输出层** | templates/ | 标准化报告格式 |
| **推送层** | scripts/push-*.py | 飞书 + 邮件双通道 |
| **持久层** | intelligence-reports/ | 本地 + Git 备份存储 |

## 4. 技能体系

### 4.1 主控技能

```
xfusion-intelligence (SKILL.md)
├── 定义5大监控领域
├── 定义5个采集代理
├── 定义报告模板
└── 定义推送流程
```

### 4.2 子技能（可独立触发）

```
sub-skills/
├── power-supply-deep/     → 电源行业深度
├── liquid-cooling-deep/   → 液冷行业深度
├── ai-software-deep/      → AI软件与方案深度
├── space-computing-deep/  → 太空算力深度
└── hr-intelligence-deep/  → 人力资源深度
```

每个子技能包含：
- 该领域的完整监控企业清单
- 专用搜索关键词和策略
- 深度报告模板
- 独立触发条件（可用 `skill_view()` 加载后执行）

### 4.3 技能间关系

```
主控周报 ──汇总──> 5个子领域各输出3-5条核心信号
子领域深度 ──纳入──> 主控周报的跨周期分析基线
主控周报 ──触发──> 子领域深度（异常信号、月度深度）
```

## 5. 推送体系

| 通道 | 格式 | 字数 | 触发条件 | 实现 |
|------|------|------|---------|------|
| 飞书 | 纯文本 | ≤200字 | 每次周报 | `push-feishu.py` + Webhook |
| 邮件 | HTML | 完整 | 每次周报 | `push-email.py` + SMTP |
| Git备份 | Markdown | 原始 | 每次生成 | `backup.sh` |

## 6. 扩展指南

### 新增监控方向

1. 在 `references/company-watchlist.md` 中添加新企业和关键词
2. 在 `SKILL.md` 中添加新代理定义
3. 在 `sub-skills/` 下创建对应的子技能
4. 更新报告模板添加新章节

### 新增推送通道

1. 在 `scripts/` 下创建推送脚本（如 `push-wechat.py`）
2. 在 `SKILL.md` 步骤5中添加推送调用
3. 更新 `references/push-config.md`

### 调整执行频率

- 周报: 修改 cron 的 `schedule` 参数
- 子领域深度: 创建独立的月度 cron job
- 紧急推送: 使用 `cronjob(action='run')` 手动触发
